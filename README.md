# Roast Battle MVP

Real-time 1v1 roast battles built with Next.js App Router, Supabase Realtime, and WebRTC.

## Local setup

1. Install dependencies:

```bash
npm install
```

2. Create env file:

```bash
cp .env.example .env.local
```

3. Fill `.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=sb_publishable_your_public_key
```

4. Run:

```bash
npm run dev
```

## Supabase setup

Run this SQL in Supabase SQL editor:

```sql
create table if not exists public.queue (
  id text primary key,
  created_at timestamptz not null default now(),
  status text not null check (status in ('waiting', 'matched'))
);

create table if not exists public.matches (
  id bigint generated by default as identity primary key,
  created_at timestamptz not null default now(),
  user_a text not null,
  user_b text not null,
  status text not null
);

alter table public.queue enable row level security;
alter table public.matches enable row level security;

drop policy if exists "queue select anon" on public.queue;
drop policy if exists "queue insert anon" on public.queue;
drop policy if exists "queue update anon" on public.queue;
drop policy if exists "matches select anon" on public.matches;
drop policy if exists "matches insert anon" on public.matches;
drop policy if exists "matches update anon" on public.matches;

create policy "queue select anon" on public.queue for select to anon using (true);
create policy "queue insert anon" on public.queue for insert to anon with check (true);
create policy "queue update anon" on public.queue for update to anon using (true) with check (true);
create policy "matches select anon" on public.matches for select to anon using (true);
create policy "matches insert anon" on public.matches for insert to anon with check (true);
create policy "matches update anon" on public.matches for update to anon using (true) with check (true);

alter publication supabase_realtime add table public.matches;
```

## Vercel deployment

1. Import this repo into Vercel.
2. Add these environment variables in Vercel project settings:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
3. Deploy.

The app is static/serverless friendly and should deploy with the default Next.js settings.
